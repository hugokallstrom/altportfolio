Repli

An idea that originated at Umeå University but continued to be developed outside of school. The purpose of the application is to connect people around the world, but with images instead of text messages. There are no friend lists and no way to search for users, instead images are served to the user instantaneous after a picture is taken. The user can then select which picture to respond to and then begin an "image only"-conversation. Repli is running on Google App Engine for scalability and currently supports Android as the mobile platform.

ReadyApp

Another mobile application developed at Umeå University for the Advanced Distributed Systems course. The purpose of the app is to let the user organize events in a fast and convenient way. There are a few ways of organizing friends and/or colleagues for an event, the most popular probably being facebook. But sometimes the event can be something not "worthy" for a facebook event, perhaps it only involves 6-7 people and starts in 1 hour.

ReadyApp makes it very convenient to create an event, invite people from a contact list and initate a ready check. The ready check will send a notification to every attendee in the event which they can respond ready or not ready to. This makes organizing small activities or events a breeze.

Genomizer

The course Software Development Techniques gave the oppurtunity to develop a fully functional system for the research group EpiCoN, using modern methods such as Scrum to mimic a "real world" project. Genomizer is a system for storing and analyzing data related to DNA research. There is extensive documentation on Github where all details can be found. The documentation and code needed to be very clear and understandable since new students will continue developing the system every year. I worked with the communications group in this project, which meant connecting clients (web, iOS, Android and desktop) with the server using a RESTful API (which I created!).
